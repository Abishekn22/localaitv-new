## Cross-laptop usage (macOS → macOS)

### On the second laptop, first time
1. Install: Git (built-in), Node, Chrome, Netlify CLI (`npm i -g netlify-cli`), optionally GitHub CLI (`brew install gh`).
2. Clone this repo:
   ```bash
   git clone https://github.com/nagarjunak-pixel/localaitv-app-frontend.git ~/localaitv-app-repo
   cd ~/localaitv-app-repo
   ```
3. Open the repo in Claude Code. In your first message, say:
   > "Read `planning/CONTEXT_SUMMARY.md` and `planning/claude-memory/project_admin_dashboard_pipeline.md` to orient yourself before we continue."
4. Copy the memory files into the second laptop's Claude memory location (Claude Code creates the project-specific folder under `~/.claude/projects/<auto-generated-slug>/memory/` on first run — wait for it to be created, then copy):
   ```bash
   # After first Claude Code session opens, the folder exists. Then:
   cp ~/localaitv-app-repo/planning/claude-memory/*.md \
      ~/.claude/projects/<auto-generated-slug>/memory/
   ```
5. Netlify CLI authentication on the new laptop:
   ```bash
   netlify login   # one-time browser auth, same account
   ```

### Daily workflow (either laptop)
```bash
# Before editing — always pull first
cd ~/localaitv-app-repo
git pull origin main

# Edit App.jsx (this is the canonical source on both laptops from now on)
# Regenerate preview.html from App.jsx (use the Python splice — see CONTEXT_SUMMARY §7.3)

# Then sync to GitHub

git push origin main
```
